# 実践型問題解決チャレンジ
## ～データ探偵プロジェクト～
## 研修概要

本研修は、エンジニアとして現場で求められる「問題解決力」を養うことを目的とした実践型課題解決研修です。

実際のプロジェクトでは、
* 設計書が古い
* ドキュメントが不足している
* 顧客の認識が間違っている
* 情報が複数存在し、どれが正しいか分からない
といった状況が頻繁に発生します。

そのような状況の中で、
* 事実を集める
* 仮説を立てる
* 検証する
* チームで議論する
* 根拠を持って結論を出す
という一連の問題解決プロセスを体験していただきます。

---

# 研修目的
本研修では以下の能力向上を目指します。
正解を出すことよりも回答までに至ったプロセスを重要視します。
* 問題解決力
* 仮説思考力
* 調査力
* 情報整理力
* コミュニケーション力
* 説明力
* AI活用力
* チームワーク

---
# 研修コンセプト
皆さんはデータ分析システムの開発チームとしてプロジェクトに参加します。
運営から提供される情報は以下の通りです。
* API
* データベース
* ログ
* IF仕様書
* DB定義書
* テスト仕様書
* 顧客からの問い合わせ情報
しかし、これらの情報は必ずしも一致していません。
例えば、
* 設計書が古い
* 顧客の認識が誤っている
* テスト仕様書が更新されていない
など、現場で実際に起こり得る状況を再現しています。
皆さんのミッションは、様々な情報から事実を見つけ出し、正解ではなく真実と課題を導き出すことです。

---

# 基本ルール
## 真実は必ず存在する
本研修には必ず正解があります。
運や偶然で解決する問題は出題しません。
情報を収集し、整理し、正しく検証すれば必ず真因へ辿り着けます。

---

## 情報は一致しない
全ての資料が正しいとは限りません。
どの情報を信頼するべきかを判断することも研修の一部です。

## 環境構築・起動手順

Python 3.11以降を想定しています。

### Pythonのインストール

1. [Python公式サイト](https://www.python.org/downloads/) から最新版のインストーラをダウンロードする
2. インストーラ実行時に画面下部の **「Add python.exe to PATH」にチェックを入れて**インストールする
3. インストール後、ターミナルを開き直して以下を実行し、バージョンが表示されることを確認する
   ```powershell
   python --version
   ```

#### `python`コマンドが見つからない場合（環境変数PATHの設定）

インストール時にPATHへの追加チェックを入れ忘れた場合などは、手動で環境変数に追加します。

1. Win+R → `sysdm.cpl` と入力してEnter →「詳細設定」タブ →「環境変数」を開く
2. 「ユーザー環境変数」の `Path` を選択して「編集」し、以下の2つのパスを追加する（パスはインストール時の選択により異なります）
   - `C:\Users\<ユーザー名>\AppData\Local\Programs\Python\Python3xx\`
   - `C:\Users\<ユーザー名>\AppData\Local\Programs\Python\Python3xx\Scripts\`
3. ターミナルを開き直して `python --version` で確認する

### 仮想環境の作成

```powershell
cd pandora
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### DBの初期化

```powershell
python data/seed.py
```

`data/pandora.db` と `data/logs/` 配下のログファイルが生成されます。

### DBの内容確認

別途インストールしなくても、Pythonに同梱の `sqlite3` モジュールでDBを開けます。

```powershell
python -m sqlite3 data/pandora.db
```

起動後、以下のSQL文で各テーブルの内容を確認できます。

```sql
-- 広告主マスタ
SELECT * FROM advertisers;

-- 代理店マスタ
SELECT * FROM agencies;

-- 媒体社マスタ
SELECT * FROM publishers;

-- キャンペーン
SELECT * FROM campaigns;

-- 日次配信実績
SELECT * FROM delivery_daily;

-- 請求
SELECT * FROM invoices;
```

### APIサーバーの起動

```powershell
uvicorn app.main:app --reload --reload-dir app
```

`--reload-dir app` を付けることで、再読み込みの監視対象を `app/` ディレクトリ配下のみに限定しています。
これを付けない場合、`--reload`はカレントディレクトリ全体（`data/`のDBファイルや`docs/`のドキュメントなど）を監視するため、
APIのコードと無関係な変更でも再起動が走り、ターミナルに `WatchFiles detected changes in ...` が頻繁に表示されてしまいます。

起動後、`http://127.0.0.1:8000/docs` でSwagger UI（API仕様の確認・実行）が利用できます。

### テストの実行

`tests/` 配下に `docs/test_spec.md` のテストケース(TC-01〜TC-04)に対応する単体テストがあります。

```powershell
pytest
```

テスト実行時にDBが初期状態へ再構築されるため、事前の `python data/seed.py` 実行は不要です。

## 主なAPIエンドポイント

| メソッド | パス | 概要 |
| --- | --- | --- |
| GET | `/advertisers`, `/advertisers/{id}` | 広告主マスタ |
| GET | `/agencies`, `/agencies/{id}` | 代理店マスタ |
| GET | `/publishers`, `/publishers/{id}` | 媒体社マスタ |
| GET | `/campaigns`, `/campaigns/{id}` | キャンペーン |
| GET | `/campaigns/{id}/delivery?from=&to=` | 日次配信実績 |
| GET | `/campaigns/{id}/invoices`, `/invoices/{id}` | 請求情報 |

## Swagger UIでのレスポンス確認方法

APIサーバー起動後、ブラウザで `http://127.0.0.1:8000/docs` を開くとSwagger UIが表示されます。

1. 確認したいエンドポイント（例: `GET /campaigns/{campaign_id}/delivery`）をクリックして展開する
2. 右側の **「Try it out」** をクリックする
3. パスパラメータ・クエリパラメータ（例: `campaign_id=C001`, `from=2026-05-01`, `to=2026-05-31`）を入力する
4. **「Execute」** をクリックして実行する
5. 画面下部の **「Responses」** に、実際のレスポンスステータス・レスポンスボディ（JSON）・curlコマンドが表示される

パラメータを変えながら何度でも実行できるため、複数キャンペーンのレスポンスを見比べる際にも利用できます。
